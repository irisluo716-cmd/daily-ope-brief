# 独立工程 · 电商评论 VoC 采集（Amazon / Home Depot / Lowe's）

> 立档：2026-06-23　状态：**独立工程 · 待启动**（不在每日简报当前迭代内做，后续单独拉出来做）
> 归属：每日简报 backlog 的 B3 拆出；产物回流 `资产库/对手SKU库`、`资产库/用户场景库`、`资产库/用户画像库`。
> 目的：把"用户原声 / 真实痛点 / 评分分布 / 真实使用细节（如实际功率、续航）"接入系统，补上目前完全缺失、但价值最高的一类信息。

---

## 1. 死结
三家都没有"公开评论 API"：
- **Amazon**：PA-API（联盟接口）只给价格/图片/评分数量，**不返回评论正文**。
- **Home Depot / Lowe's**：无公开商品/评论 API。

→ 评论正文只能 **①第三方替你爬（托管数据服务）** 或 **②自己爬**；另有一条 HD/Lowe's 专属巧路（Bazaarvoice）。

## 2. 四条路线对比

| 路线 | 覆盖 | 稳定性 | 成本 | 维护 | 法律/ToS | 适合 |
|---|---|---|---|---|---|---|
| **A. 第三方评论 API**（托管爬虫） | 三家 | 高（厂商扛反爬） | 按结果计费 | 几乎零 | 灰，风险厂商担 | 日常自动监控 ⭐ |
| **B. Bazaarvoice 直连** | 仅 HD/Lowe's | 中高（半官方接口） | 极低 | 低 | 灰 | HD/Lowe's 省钱 |
| **C. Browser MCP 人工**（住宅 IP） | 三家 | 高 | $0 | 中 | 较安全 | 立项深挖（大样本）⭐ |
| **D. 自建爬虫+代理** | 三家 | 低 | 代理费 | 高 | 风险自担 | ❌ 不建议（维护坑） |

**A 的候选厂商**（启动时需核现行覆盖/价格）：Unwrangle（**同时覆盖三家**，最匹配）、Rainforest API（专 Amazon）、Apify（三家各有 actor，按量付费）、Oxylabs / Bright Data（企业向）、SerpApi（有 Home Depot，**无 Amazon**）。成本量级：只盯重点 SKU、每周拉近评，**约每月几十美元级**［估算，以厂商现行报价为准］。

**B 的原理**：HD/Lowe's 评论都跑在 **Bazaarvoice** 平台上；商品页源码里有 passkey + 商品外部 ID，直接调 `api.bazaarvoice.com/data/reviews.json` 取结构化评论，比抓 HTML 稳、几乎免费。Amazon 不走 Bazaarvoice。

## 3. 推荐组合（广度 + 深度分开）
- **日常监控（广度）**：A 里选一个覆盖三家的（如 Unwrangle），只盯 `对手SKU库` 重点 SKU，每周拉近评 → 进系统。低成本、稳、GitHub Actions 可调。
- **立项深挖（深度）**：继续 C（Browser MCP 人工），大样本扒透。
- **想再省**：HD/Lowe's 走 B（Bazaarvoice），Amazon 用 A。

## 4. 接入架构（和 vault 打通）
```
对手SKU库（重点 SKU 清单）→ 解析 ASIN / 门店商品ID → 第三方API拉近评
→ MiniMax 提炼「评分分布 + 痛点 Top + 真实使用细节」
→ 标【样本量 × 置信度】→ 回写 用户场景库 / 用户画像库
```
天然带样本量，符合推测标注规则；接上"真实功率分布 / 用户原话"那套。

## 5. 三个必须认的坑
1. **成本**：第三方 API 是唯一持续花钱处（其余基本免费）；只盯重点 SKU 量不大，可控。
2. **ToS / 法律**：评论是公开内容但抓取违反平台 ToS——用第三方（风险它担）、只取样本不批量搬库、不商用转售。
3. **数据噪音**：刷评/水军不少——别只看星级，看长评、看 verified purchase、看差评里的具体故障；用户名一律打码（CI 规则）。
